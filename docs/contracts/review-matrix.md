## Review Rules

- Every source claim that drives an SDK primitive must link to a primary source.
- Every source-backed row should state the source lesson, the primitive change, and the validation gate it affects.
- Every implemented event kind must have a golden payload fixture before an adapter emits it.
- Every mutating side effect must have intent and result records or be marked non-reversible.
- Product-specific behavior must not enter active SDK contracts. If a host needs something new, map it to a product-neutral SDK primitive or keep it outside core.
- Any mismatch found during coding updates the relevant contract before code changes continue.
- Any new capability variant must name its typed sidecar contract, owner role, fingerprint fields, emitted events, journal records, and acceptance tests before an adapter can emit or execute it.
